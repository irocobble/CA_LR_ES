# 1. KiCad

The hardware for OpenEdge-S3 is designed using **KiCad 9**.

## Opening the Project

1. Install **KiCad 9** or a newer compatible version.
2. Clone this repository.

```bash
git clone <repository-url>
```

3. Open KiCad.

4. Open the project file located in:

```text
PCB/OpenEdge-S3.kicad_pro
```

*(Replace the filename if the project name changes.)*

---

## Project Structure

```text
PCB/
├── OpenEdge-S3.kicad_pro
├── OpenEdge-S3.kicad_sch
├── OpenEdge-S3.kicad_pcb
├── Libraries/
├── Footprints/
├── Symbols/
└── Manufacturing/
```

---

## Editing the Schematic

Open:

```
Schematic Editor
```

Here you can

- Modify the schematic
- Add components

Open:

## Manufacturing Files

Generate the following before fabrication:

# 2. ESP-IDF

OpenEdge-S3 firmware is primarily developed using **ESP-IDF**.

---

## Installing ESP-IDF

### Step 1 — Create a Workspace

```bash
mkdir -p ~/esp
```

Move into the workspace.

```bash
cd ~/esp
```

---

### Step 2 — Clone ESP-IDF

Clone the official ESP-IDF repository.

```bash
git clone --recursive https://github.com/espressif/esp-idf.git
```

Move into the ESP-IDF directory.

```bash
cd esp-idf
```

---

### Step 3 — Install ESP-IDF

Run the installation script.

```bash
./install.sh
```

This installs the required toolchains and dependencies.

---

# Activating the Environment

Before building any project, activate the ESP-IDF environment.

```bash
source export.sh
```

> **Note:** This command must be executed every time a new terminal is opened.

---

# Creating a New Project

Move back to the workspace.

```bash
cd ~/esp
```

Create a new project.

```bash
idf.py create-project <project_name>
```

Move into the project directory.

```bash
cd <project_name>
```

---

# Project Structure (Important)

Your project should follow the structure below.

```text
my_project/
│
├── CMakeLists.txt
├── sdkconfig
│
└── main/
    ├── CMakeLists.txt
    └── main.c
```

### Description

**CMakeLists.txt**

Contains the project build configuration.

**sdkconfig**

Stores all ESP-IDF configuration generated by Menuconfig.

**main/**

Contains your application source code.

> **Important:**  
> The project structure must remain exactly like this. Incorrect folder organization can cause build failures and unnecessary debugging headaches.

---

# Menuconfig (The Heart of ESP-IDF)

Open Menuconfig using

```bash
idf.py menuconfig
```

Menuconfig allows configuration of almost every aspect of the ESP32.

---

## Common Configuration

For this project, recommended settings include:

```
Component config
    └── Wi-Fi
          └── Enable CSI
```

Configure your Wi-Fi credentials under the project-specific configuration if required.

Example:

```
CSI Wi-Fi Settings

SSID

Password
```

---

# Building the Project

### Step 1

Activate ESP-IDF.

```bash
source export.sh
```

---

### Step 2

Navigate to the project directory.

```bash
cd ~/esp/<project_name>
```

---

### Step 3

Build the project.

```bash
idf.py build
```

---

### Step 4

Flash the firmware.

```bash
idf.py -p <PORT> flash
```

Example:

```bash
idf.py -p /dev/ttyUSB0 flash
```

or

```bash
idf.py -p COM5 flash
```

---

### Step 5

Open the serial monitor.

```bash
idf.py monitor
```

To exit the monitor:

```
Ctrl + ]
```

---

# Useful ESP-IDF Commands

| Command | Description |
|----------|-------------|
| `idf.py build` | Build project |
| `idf.py flash` | Flash firmware |
| `idf.py monitor` | Open serial monitor |
| `idf.py menuconfig` | Configure project |
| `idf.py erase-flash` | Erase flash memory |
| `idf.py fullclean` | Clean build directory |
